# 宣教実績ダッシュボード ― 設計書

対象読者：全国約30教会の責任者。前年比・地域比較・規模比較を通じて宣教実績を把握する。

---

## 1. 画面設計

### 1.1 画面一覧

| # | 画面 | URL（Hashルーティング） | 概要 |
|---|------|--------------------------|------|
| 1 | ダッシュボード（トップ） | `#/` | サマリー・ハイライト・グラフ・スコアカード一覧 |
| 2 | 教会詳細 | `#/church/:id` | 1教会の全KPI詳細・比較・分析コメント |
| 3 | 404 | 上記以外 | 存在しないパスへのフォールバック |

GitHub Pages はサーバー側のルーティングを持たない静的ホスティングのため、`BrowserRouter` ではなく `HashRouter` を採用し、`404.html` のリライトトリックなしで直接リンク・リロードに対応する。

### 1.5 スコアカードの状態遷移

カードの背景色は「指標切替」の選択に応じて3パターンに変化する：

| 指標 | 配色ルール |
|---|---|
| 2026年実績 | 人数に応じた5段階グラデーション（アクセントカラーの濃淡） |
| 前年差 | プラス=緑 / ゼロ=グレー / マイナス=赤 |
| 前年比 | 120%以上=濃い緑／100〜119%=薄い緑／100%=グレー／80〜99%=薄い赤／80%未満=濃い赤 |

KPI・表示単位（全体/男性/女性）・指標の3つの切替はすべて画面全体（サマリー・ハイライト・グラフ・カード）に即時反映される（グローバル状態）。

## 2. UIデザイン方針

**方向性**：GitHub / Linear / Notion / Vercel Dashboard を参照した、装飾を抑えたプロダクト然としたデザイン。

- **配色**：ライトモードは白〜オフホワイト背景（`#FAFAFA`）＋濃紺インディゴのアクセント（`#4F5DED`）。ダークモードは`#0A0A0B`背景。ステータスカラー（緑=増加・赤=減少・グレー=横ばい）はどちらのモードでも彩度を保ったまま使用。
- **タイポグラフィ**：UI本文は Inter（和文フォールバックに Noto Sans JP）。数値（人数・％）は等幅数字が扱える `tabular-nums` を指定し、桁の変化が視覚的に揺れないようにする。
- **カード**：角丸 `0.875rem`、薄いボーダー＋控えめなシャドウ。ホバーで僅かに浮き上がる。
- **密度**：教会責任者が並べて比較する画面のため、余白を切り詰めた高密度レイアウトを採用しつつ、タップ領域は44px以上を確保。
- **レスポンシブ**：カード一覧は `1列(モバイル) → 2列(タブレット) → 3〜4列(デスクトップ)`。グラフも `1列 → 2列 → 3〜4列` にリフロー。
- **ダークモード**：`class`戦略。トグルボタンで即時切替、`localStorage`に保存し次回訪問時も復元。
- **アクセシビリティ**：フォーカスリング常時表示、`prefers-reduced-motion`尊重。

## 3. システム設計

### 3.1 全体構成

```
┌─────────────────────────────────────────────┐
│               ブラウザ（クライアント）         │
│  ┌───────────────────────────────────────┐  │
│  │  React SPA (Vite build 済み静的アセット) │  │
│  │  ├─ DashboardContext（グローバル状態）    │  │
│  │  ├─ Pages（Dashboard / ChurchDetail）    │  │
│  │  ├─ Components（Cards/Charts/Filters）   │  │
│  │  └─ churches.json（ビルド時にバンドル）   │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
                     ▲
                     │ HTTPS（静的ファイル配信のみ）
                     │
            GitHub Pages（サーバー・DBなし）
                     ▲
                     │ push → GitHub Actions が自動ビルド・デプロイ
                     │
              GitHub リポジトリ（main branch）
```

- サーバーサイドの処理・API・データベースは一切使用しない。
- 教会実績データは**ビルド時**にJSONとしてバンドルされ、実行時は完全にクライアントサイドで集計・描画する。
- データ更新のたびに Excel → JSON 変換 → コミット → push（GitHub Actionsが自動でビルド・再デプロイ）というフローになる。

### 3.2 技術スタック

| レイヤー | 採用技術 | 理由 |
|---|---|---|
| ビルドツール | Vite | 高速な開発体験・シンプルな静的ビルド出力 |
| 言語 | TypeScript | データ構造（KPI・性別・指標）を型で保証し、集計ロジックのバグを防止 |
| UI | React 18 | コンポーネント指向でカード・グラフ・フィルターの再利用性が高い |
| スタイリング | Tailwind CSS | ダークモード・レスポンシブ・カラースケールをユーティリティで一元管理 |
| グラフ | Recharts | React親和性が高く、8種のグラフを宣言的に実装可能 |
| ルーティング | react-router-dom（HashRouter） | GitHub Pagesでサーバーリライト不要 |
| ホスティング | GitHub Pages | 追加コストなし。GitHub Actionsでpush時に自動デプロイ |
| 検索避け | `robots.txt`（Disallow: /）＋ `<meta name="robots" content="noindex, nofollow">` | 関係者限定ポータルを検索結果から除外 |

### 3.3 状態管理

`DashboardContext`（React Context + useState）でグローバル状態を一元管理する。サーバーを持たないため、Redux等の外部ライブラリは不要と判断。

保持する状態：
- `kpi`（選択中のKPI）
- `gender`（表示単位：全体／男性／女性）
- `metricMode`（指標：実績／前年差／前年比）
- `regionFilter` / `sizeFilter`（複数選択フィルター）
- `sortKey`（並び替え条件）
- `theme`（ライト／ダーク、localStorage永続化）

これらの状態はダッシュボード全体（サマリー・ハイライト・グラフ・カード一覧）で共有され、切替時に全コンポーネントが再計算・再描画される。

### 3.4 デプロイフロー（GitHub Actions）

`.github/workflows/deploy.yml` にて、`main` ブランチへの push をトリガーに以下を自動実行：

1. `npm ci` で依存関係をインストール
2. `npm run build` で `dist/` に静的ファイルを生成
3. `actions/upload-pages-artifact` → `actions/deploy-pages` で GitHub Pages に公開

手動でのビルド・デプロイ作業は不要になる。

## 4. データ設計

### 4.1 元データ（Excel）

| シート | 内容 |
|---|---|
| 教会実績 | 教会名（番号付）／年（2025・2026）／性別（男・女）／KPI（BS開始・デビュー・BSストップ・BS中）／人数 のロング形式、計480行 |
| 教会マスタ | 教会名（番号付）／正式教会名／総メンバー数／教会規模／地域、30行 |

### 4.2 変換後データ（`src/data/churches.json`）

Excelのロング形式データを、教会単位でKPI×性別×年をネストした構造に変換し、ビルド時にJSとしてバンドルする（実行時のAPI呼び出し・DB結合は発生しない）。

```jsonc
[
  {
    "id": "01",
    "key": "01_主新和",
    "name": "主新和",
    "region": "東日本",
    "size": "大規模",
    "totalMembers": 315,
    "kpi": {
      "BS開始": {
        "total": { "y2025": 22, "y2026": 37 },
        "男":    { "y2025": 9,  "y2026": 18 },
        "女":    { "y2025": 13, "y2026": 19 }
      },
      "デビュー": { "total": {...}, "男": {...}, "女": {...} },
      "BSストップ": { "total": {...}, "男": {...}, "女": {...} },
      "BS中": { "total": {...}, "男": {...}, "女": {...} }
    }
  }
  // ... 全30教会
]
```

`total`（全体）は男女の合算値としてPython変換スクリプト側で事前計算済み。

### 4.3 計算ロジック（`src/lib/calc.ts`）

| 指標 | 計算式 |
|---|---|
| 前年差 | `2026年実績 − 2025年実績` |
| 前年比 | `2026年実績 ÷ 2025年実績 × 100`（小数1桁表示、2025年実績が0の場合は「算出不可」として`null`） |
| 教会単位の平均前年比 | 各教会の前年比を単純平均（前年比算出不可の教会は母数から除外） |
| 全国／地域／規模平均 | 対象教会群の合計値から前年比・前年差を算出（グラフ）／各教会の前年比を単純平均（教会詳細の比較表） |

### 4.4 データ変換スクリプト

`scripts/convert_excel.py`（Python・openpyxl使用）で `前年比.xlsx` → `src/data/churches.json` を再生成できる。データ更新時は本スクリプトを再実行してコミットするだけでよい。

```
python3 scripts/convert_excel.py 前年比.xlsx src/data
```

## 5. 実装手順

1. **データ変換**：`scripts/convert_excel.py` で Excel を `churches.json` に変換（実施済み・同梱）
2. **プロジェクト基盤**：Vite + React + TypeScript + Tailwind の設定一式（`vite.config.ts` / `tailwind.config.js` / `tsconfig.json`）
3. **型・計算ロジック**：`src/types.ts`（KPI・性別・指標の型定義）、`src/lib/calc.ts`（前年比・前年差・平均の算出）、`src/lib/colorScale.ts`（3種の色分けルール）、`src/lib/insights.ts`（分析コメント自動生成）
4. **グローバル状態**：`src/context/DashboardContext.tsx`
5. **共通コンポーネント**：Header／SegmentedControl／ControlBar／SummaryCards／ScoreCard／Highlights／FilterBar／ScoreCardGrid
6. **グラフ実装**：`src/components/charts/` 以下に8種のグラフ（RankingChart×3、GroupAverageChart×4、KpiCompareChart×1）
7. **ページ実装**：DashboardPage／ChurchDetailPage／NotFoundPage、`react-router-dom`（HashRouter）でルーティング
8. **検索避け対応**：`public/robots.txt`（`Disallow: /`）、`index.html`の`<meta name="robots">`
9. **GitHub Actions**：`.github/workflows/deploy.yml` で push時の自動ビルド・GitHub Pagesデプロイ
10. **公開**：GitHubリポジトリ作成 → push → Settings → Pages で「GitHub Actions」をソースに指定 → 数分後に公開URL発行

デプロイ手順の詳細は `README.md` を参照。
